Compute Scheduler

Overview

The compute server scheduler (CSS) allows to share our compute servers in a fine-grained manner. Users can request their commands to run exclusively. Other users trying the same will be blocked. (It is not actually a scheduler, just an allocator).

All time-consuming commands on such machines must be preceded by either 'exclusive' or 'nonexclusive'.

Usage

The two main commands are 
css
(type css to see a list of subcommands) and 
exclusive

To run a command exclusively, type 
exclusive [command]

To reserve the computer for your own for a while, start an exclusive shell or screen: 
exclusive bash
exclusive screen

If the system is locked up because one of your command crashed badly, type 
css clear
Only do this if you are sure that no other exclusive or nonexclusive commands are running. 
css info

shows the current status of the machine, while 
css overview

shows an overview of the load of all machines.

If you do not need a machine exclusively, but your work would interfere with exclusive jobs, please start your program using 
nonexclusive [command]

Multiple users can share a machine non-exclusively, but there can be only one exclusive job.

If you just need a compute node, and do not know which ones are currently free, login in to aeghpc105, which is a shared always free ssh-entry server that can also be used for simple compute tasks. 
ssh aeghpc105.ifi.uni-heidelberg.de
